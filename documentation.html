Autentikasi

Semua endpoint yang terdaftar di sini memerlukan autentikasi. Ada dua metode yang didukung:

1. Pengguna Terdaftar (Logged-in)

Kirim header Authorization dengan ID Token yang didapat dari Firebase Authentication setelah pengguna login.

Contoh Header: Authorization: Bearer [ID_TOKEN_PENGGUNA]

2. Tamu (Guest)

Tamu dapat menggunakan API dengan batasan. Setiap request dari tamu harus menyertakan header x-device-id yang berisi ID unik untuk perangkatnya (misalnya UUID yang dibuat di sisi klien).

  • Header Wajib: x-device-id
  • Batas Penggunaan: Tamu hanya dapat melakukan 5 kali permintaan analisis atau chat.
  • Jika Batas Terlampaui: Server akan merespons dengan status code 429 Too Many Requests.

Contoh Header: x-device-id: 123e4567-e89b-12d3-a456-426614174000

POST /api/analyze

Mengunggah dan menganalisis dokumen (PDF/DOCX). Endpoint ini akan memulai sesi chat baru.

Full URL: {{BASE_URL}}/api/analyze

Body: multipart/form-data dengan field document berisi file.

POST /api/chat/continue/:chatId

Mengirimkan pertanyaan lanjutan dalam sebuah sesi chat yang sudah ada.

Full URL: {{BASE_URL}}/api/chat/continue/<chatId>

Body: JSON { "userQuestion": "Isi pertanyaan Anda di sini" }

GET /api/chats

Mengambil daftar semua riwayat percakapan (chat history) milik pengguna.

Full URL: {{BASE_URL}}/api/chats

GET /api/chats/:chatId

Mengambil detail pesan dari satu percakapan spesifik.

Full URL: {{BASE_URL}}/api/chats/<chatId>

DELETE /api/chats/:chatId

Menghapus satu percakapan spesifik dari riwayat.

Full URL: {{BASE_URL}}/api/chats/<chatId>

DELETE /api/chats

Menghapus semua riwayat percakapan milik pengguna.

Full URL: {{BASE_URL}}/api/chats
